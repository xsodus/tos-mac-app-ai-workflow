# Publication Checklist

This repository has been repositioned for public sharing as a reusable AI
workflow automation framework.

## Publishing Goals

1. Present the project as desktop automation and loop engineering.
2. Keep domain-specific examples separate from the core framework.
3. Remove personal paths and machine-specific references.
4. Keep documentation aligned with MIT licensing and public open-source norms.

## Audit Notes

- Replace absolute paths like `./relative/path`.
- Keep the core README focused on skills, controller design, and safety.
- Preserve domain-specific examples only under `examples/`.
- Verify Terms-of-Service guidance is visible in documentation.
